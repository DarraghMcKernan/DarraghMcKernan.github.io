Usability Assesment

Consistency and Standards

  • Good Examples
  • When in the main menu it shows all options and information around the centre
    • of the page keeping them all spread out and easy to distinguish.
  • The other menus can all still be accessed from any menu making it very quick
    • and easy to navigate the website.
  • Bad Examples
  • If you are in a teams call there are no labels on most buttons unless the user
    • hovers over the button which they might not know is actually there.

Visibility of System status

  • Good Examples
  • When uploading a file a progress bar shows how far along the upload is and will
    • show a checkmark when the upload is finished.
  • When in a menu its icon is highlighted a blue colour and a blue line is shown
    • beside it.
  • Bad Examples
  • The calender menu is too simple and could display more information if the space
    • was used more effiecently by making the days and hours grid smaller.

User control and freedom

  • Good Examples
  • The button for hanging up a teams call is a bright red that stands out against
    • the blues and purples that teams typically uses.
  • Bad Examples
  • It is not as clear to cancel a file that is uploading and it could possibly be
    • made red too.

Error Prevention

  • Good Examples
  • When joining a call a screen pops up telling you if your microphone is muted and
    • if your camera is on or off.
  • Bad Examples
  • When pressing hang up it will instantly leave the call. Having a simple pop up saying
    • 'Are you sure you want to leave the call' would prevent accidental hang ups.

Recognition rather than Recall

  • Good Examples
  • Clear obvious buttons for hanging up.
  • Menus have recognisable icons and are labelled clearly.
  • Bad Examples
  • The apps page is filled with loads of information and it can be confusing to navigate as
    • the layout is not sorted very well.

Flexibility and efficiency of use

  • Good Examples
  • All of the 'teams' can be picked up and plced anywhere the user wants which allows users to
    • put more important teams first.
  • When using the apps menu there is a recently used apps drop down menu that is very useful for
    • users that frequently use the apps feature.
  • Bad Examples
  • It can be slow getiing to fiels inside iof assignments as there are several options that he user
    • will need to navigate to get through each menu.

Aesthetic and Minimalist Design

  • Good Examples
  • Entire main menu has very few and clearly labelled buttons making it incredibly easy to
    • navigate the site.
  • Bad Examples
  • The apps page has far too many options and is not clear enough and also has a second search
    • that is erperate from the main sites search bar which is confusing.
